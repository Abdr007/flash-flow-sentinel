Defense Stack — three layers, every alarm proven on-chain

—

The sentinel is built as three layers. Layer 1 detects attacks. Layer 2 continuously proves the vaults' integrity every cycle. Layer 3 turns a proven drain into an instant automated response — and deliberately holds no pause key of its own (it proves and signals; Flash's own authority acts). Every status below is read live; nothing here is synthetic. Each check is tagged PROOF — verified on-chain before it can fire, so it cannot false-alarm on an exploit — or RISK LIMIT — a heads-up when a configured threshold is crossed, even if the activity turns out legitimate. Only proofs can trigger the Layer 3 auto-response; risk-limits only ever notify a human.

Oracle Guard — FLASH6 Mark vs Pyth

—

A manipulated or stale price feed is the classic vault-drain vector. Every custody's on-chain CustomOracle mark (with its on-chain publish_time) is cross-checked against Pyth Lazer — directly (token-gated, via LAZER_ACCESS_TOKEN) or via the Flash V2 API Lazer feed (flashapi.trade/prices, official Lazer ids from /tokens). This compares the value written on-chain against a live read of the Lazer feed, so a mark forged at the on-chain oracle (rather than at Lazer itself) diverges immediately. Paused/closed markets (marketSession + on-chain publish_time) are shown as MARKET IDLE and never alarm. The active source is on the tag.
